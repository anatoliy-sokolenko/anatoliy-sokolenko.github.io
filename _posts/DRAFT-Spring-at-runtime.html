Any modern application ment to be configurable. There many reason for this:
  • difference between environments (dev, testing, staging, production)
  • customization of application functionality "on fly"
  • simlification of functionality extention
  • etc
Often application should be able to change behaviour at runtime.
Today I would like to cover the question of customizing Spring Framework based application functionality at runtime.
In general Spring Framework provides three different approaches to change application functionality at runtime:
  • FactoryBean
  • @Configuration
  • environment profile which will be availiable in Spring 3.1
FactoryBean
@Configuration
Environment profile
