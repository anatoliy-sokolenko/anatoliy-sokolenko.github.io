Результаты небольшого сравнения того как наиболее популярные LDAP сервера поддерживают динамическую загрузку схем.
1. Active Directory
Две MMC оснастки, обеспечивающие низкоуровневый доступ к Active Directory по LDAP:
schmmgmt.msc — управление схемами (входит в Windows Server Administration Tools Pack);
adsiedit.msc — низкоуровневое управление LDAP (входит в Windows Server 2008 Remote Server Administration Tools и Windows Server 2003 Service Pack 2 32-bit Support Tools).
И одна консольная приблуда для поддержки LDIF:
ldifde.exe — идет в нагрузку к доменному контроллеру

2. OpenDS
Управление осуществляется по средствам расширения формата LDIF:
То есть возможна загрузка из Java с помощью библиотеки JLDAP.
Кроме того родная Control Panel OpenDS позволяет вполне не плохо управлять схемами.

3. OpenLDAP
Самая кривая реализация — собственный формат, который впрочем близок к формату OpenDS, но заюзать LDIF для него не получиться
